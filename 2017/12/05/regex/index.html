Regex

Published: Tue 05 December 2017
By frank

In Uncategorized.

For some reason I always avoided having to learn regular expressions. They seemed so daunting I never felt like diving into the topic. Until now.

I wanted to replaced a dictionary with a class in a Python code base. This was long overdue, so a lot of code had to be rewritten to access the new class instead of the dictionary. The task was to go through all code and replace them all. For example self.items['eggs'] should become self.items.eggs.

PyCharm is pretty amazing but as far as I know it doesn't have a button to do this refactoring challenge automatically. So I got myself together and started learning regex. Luckily it's not as complicated as I feared. And there are some great tools online to try it out, like regexr.com.

Here is the code to find all dictionary calls:
(self\.items)\[\'(\w*)\'\]
The parentheses form a 'capturing group', we can use these later when defining the replacement content. A backslash is used to escape a character, so for example \. just means a literal dot. The part with \w* means a word group with any length.

Now replace the entries found with the following, using the two capturing groups:
$1.$2
This results in the content of the two capturing groups divided by a dot, for example self.items.eggs.

You can easily try this out on regexr.com. It can also explain why it hits on certain things, very helpful and not at all daunting.
